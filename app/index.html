Premiers pas avec Git.

Téléchargement et installation de Git.

Télécharger Git à l'adresse "http://git-scm.com/" et l'installer en suivant les indications du programme d'installation (Il suffit de cliquer sur "next" jusqu'à la dernière étape).
L'installation terminée, on ouvre le terminal avec la commande "cmd" dans la barre de recherche de Windows.
On vérifie si l'installation s'est bien passée en tapant dans la ligne de commande : git --version. Si la réponse affiche un numéro de version, c'est que Git est bien installé.

Commençons

Configuration de l'identité.

→ git config --global user.name "nom prénom".
→ git config --global user.email "identifiant@domaine.xxx".
→ git config -l.

Quelques commandes de base

→ mkdir new_project : pour créer un répertoire/dossier/projet/repository.

→ cd new_project : pour accéder/se placer sur le repository créé.

→ git init : Je dis à Git que je vais travailler sur ce "repository". Je lui demande de "versionner" mon projet.
J'obtiens comme réponse :

initialized empty git repository
in c:/users/WIN7/new_project/.git/

→ git status :
j'obtiens la réponse : il y a rien dans le projet.

on branch master
no commit yet
nothing to commit (create/copie files and use "git add" to track)
Dans c:/users/WIN7/new_project, Je crée un fichier index.html (de manière classique - avec la souris ...). je l'ouvre avec un éditeur de texte (Notepad++, Sublim Text, Brackets, ...) et j'édite juste une ligne de code, exemple : <h1>Bonjour</h1>. Puis je reviens dans Git en ligne de commande et je fais
